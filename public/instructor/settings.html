Database Management

Development Mode

Note: The database management tools below are temporary and only available during development. These features will be removed or restricted in production environments.

Reset All Data

Delete all BiocBot data including vector embeddings (Qdrant) and document metadata (MongoDB). This will completely reset the system.

Login Restrictions

Allow Local Login & Registration

When disabled, users can only log in via CWL. Local username/password login and new account registration will be hidden.

System Admin Access

System admins can manage platform-wide settings, delete all data, and grant admin access to other users.

Grant Admin Access

Enter the email for an existing user. They need to have logged in at least once before you can grant access here.

Current System Admins

Only system admins can see this list or change admin access.

Student Session Settings

Student Idle Timeout (Minutes)

Time before a student is prompted to confirm they are still active. Default is 4 minutes. The student sees a timer pop up halfway through the timeout period.

Quiz Practice Settings

Enable Quiz Practice Page

When enabled, students can access a self-paced quiz practice page. When disabled, the quiz page is hidden from the student interface.

Testable Units

Select which published units students can practice. By default, all published units are available.

Loading units...

Allow Lecture Material Access

When enabled, students can download lecture materials from units where they answered incorrectly. This helps them review and learn from mistakes.

Privacy Settings

Anonymize Students

When enabled, student names are hidden throughout your instructor dashboard. The Student Hub page is also hidden from navigation. Student counts and activity data remain visible. This setting only affects your own view.

Question Generation Settings

Customize the prompts used for AI question generation. Use placeholders: {{learningObjectives}}, {{courseMaterial}}, {{unitName}}, {{questionType}}

System Prompt

Core instructions for the AI when generating questions. Applied to all question types.

True/False Question Prompt

Template for generating true/false questions.

Multiple Choice Question Prompt

Template for generating multiple choice questions.

Short Answer Question Prompt

Template for generating short answer questions.

Mental Health Detection Settings

Configure the AI prompt used to silently detect mental health concerns in student conversations. The detection runs in parallel and does not affect the student's chat experience. Flags appear on the Flagged Content page.

Detection Prompt

This prompt instructs the AI on how to analyze conversations for mental health concerns. The AI outputs a concern level (no concern, low concern, high concern) for each student message.

Chat Retrieval Settings

Enable Source Attribution Downloads

When enabled, source citations in student chat responses become clickable download links to the underlying uploaded documents.

Additive Retrieval

When enabled, student chat for a selected unit can also use earlier published units from the course. When disabled, chat uses only the selected unit.

AI Persona Settings

Base System Prompt

Core instructions defining BiocBot's general personality and behavior.

Protege Mode Prompt

Instructions for when the AI acts as a student (Protege mode).

Tutor Mode Prompt

Instructions for when the AI acts as an instructor/tutor (Default mode).

Explain Mode Prompt

Instructions for when the AI explains a specific concept (Explain button).

Directive Mode Strategy

Instructions for how to teach when the student is struggling. The system will automatically announce the topic (e.g. "Student is struggling with X").

Quiz Help Chat Prompt

Instructions for the inline chat that appears when students answer quiz questions incorrectly. The chat helps students understand why the correct answer is right.

Course Lifecycle

Deactivate Course

When a course is deactivated, students and TAs are blocked from using it. Instructors can still open the course here and reactivate it later.

Transfer Course

Create a brand-new course copy from the current course. Materials are recopied and re-parsed, TAs keep their permissions if selected, and no students, chats, or historical activity are carried over.

Per-Unit Transfer

Learning objectives and assessment questions start on by default. Documents also bring their parsed topics along to the new course.
